# 데이터로 말해요! 데이터 중심의 사고,기획,보고의 기술

## 1. 비즈니스의 데이터
### 1-1. 디지털과 산업의 변화
#### 세계 시가 총액 top10 기업의 변화
- 세계 top10 중에서 무려 7개 기업이 IT 기업, 전통 기업들은 IT 기업처럼 바뀌기 위해 노력
  
#### 디지털의 3요소
- 디지타이제이션(Digitization): 전산화,   
  아날로그 데이터를 디지털 데이터로 전환하는 것
  비정형 데이터를 정형 데이터로 전환하는 것  
- 디지털라이제이션(Digitalization): 정보화,
  디지털 데이터를 가공하고 분석하는 것  
  데이터를 비즈니스나 업무에 활용하는 것  
- 디지털 포메이션(Digital Transformation): 디지털 혁신,
  디지털 비즈니스로 전환, 일하는 방식을 디지털로 전환

### 1-2. 비즈니스와 데이터의 관계
#### 비즈니스의 시작은 문제
- 비즈니스는 문제 해결 과정이다. 문제가 있어야 비즈니스를 시작할 수 있다.
- 그런데 문제가 있다고 해서 전부 다 비즈니스로 전환되지는 않는다. 인식이 있어야 한다.
- 문제를 인식하고 나면 이제 그 문제를 풀지 말지 의사결정을 할 수 있다.
- 의사 결정을 하고 해결을 지시하는 순간 우리에게 생기는 것은 **'목적'**이다.
- 문제가 무엇이고 어떻게 인식하고 있고 전에는 안 했는데 이번에는 왜 해결하려고 하는지를 **'배경'**이라 한다.
- 배경을 제대로 파악하고 목적을 먼저 정해야한다.
<br>
- 목적이 생기면 조사를 해야한다. 그럼 문제점이 나온다.
- 문제는 우리가 해결해야할 현상이라면, **'문제점'**은 문제를 일으키는 요인이다.
- 상사가 문제를 파악하라고 하면 그것은 문제가 아니라 문제점을 파악하라는 것이다.
- 그때 우리는 문제가 되는 데이터를 분석해서 문제점을 찾으면 된다.
<br>
- 문제점을 조사했으면 이제 분석을 하면 된다. 분석을 하면 원인 or 이유가 나온다.
- **'원인'**은 사물이나 현상에 사용하는 표현이고, **'이유'**는 사람에게 사용한다.
- **많은 직장인이 원인만 분석해서 상사에게 보고하지만 상사는 이유를더 궁금해 한다.**
- 데이터 관점에서 원인을 파악하려면 데이터를 측정하거나 분석하면 된다.
- 데이터가 명확하기 때문에 그 데이터를 측정해서 분석하는 방식을 사용하면 원인을 찾아낼 수 있다.
- 이유는 사람에게 사용한다고 했으니 당연히 사람에게 물어봐야 한다.

#### 전략, 방안, 과제, 업무
- 원인과 이유를 알아냈으면 이제 해결안을 도출한다. 그러면 **'전략'**과 **'방안'**이 나온다.
